Increase the content ratio

Keyword Ratio On a Page

Before Google turned the Web world upside down everyone mixed the styling of the page in with the content. If a designer wanted a dark blue headline using Arial font these were embedded as attributes inside the <h1> element.

Then web developers discovered that Google, and other search engines, calculated page importance by calculating how many keywords were in title, headings, and body of the page compared to the total character count.

All the sudden all those font= " " and color= " " attributes were watering down the importance of the page.

Use CSS to Separate Content from Style

CSS - Cascading Style Sheets came to the rescue by allowing web designers to put the style information for a page in the <head> element of the page. This kept it separate from the content of the page which is located inside the <body> element.

By using CSS you are able to tell the browser specifically how things should be displayed. At the same time you will not water down the content of the page.

CSS also allows you to store your CSS code in another document completely. This gives an additional value by allowing you to share the same CSS code with all the pages in the site. Need to change a style, just edit one file and every page in the site updates in appearance automatically.

Going Naked

Displaying a page without CSS code is called "going naked" and a good developer will create web pages this way.

First a designer starts with the content. The important point of the page is marked using the <h1> element. Headings that are secondary in importance are marked using the <h2> element. Each block of text is marked using either a <div> element or a <span> element. The <div> is used to mark blocks of code and the <span> element is used to mark a section of code inside another line or block of code, like special line in a paragraph.

This creates a plain-text document that uses elements to simply designate an outline showing what text is very important (the headers) and what is supportive text (the paragraphs and the alt=" " attribute for each image.).

This makes the page very accessible to non-visual browsers such as search spiders and people with visual disabilities.

Next, the CSS code is added in the <head> section of the page or in an external file.

This does not affect the search spiders but does make the page more interesting and easier to read (we hope) for the user.

How a CMS Works

New Page Info Displayed

Next, when the user sends a request to read your blog or view a CMS web page, the browser sends a request to the server, and it goes through the same process, using PHP (or another server-side language) to extract the right data from the database, builds the web page using the database information, and sends it back to the user.

So, there are two pieces.

There is the admin section where the administrator updates the database using special form pages.

The second piece is the next person that views the web page. This person will see the updated changes that were made to the database, getting an updated web page.

These are the principles that all content management systems use.

Having a Conversation with a Web Server

Client: Hello, could you give me a web page named orderBlank.html?

Server: Certainly, here it is along with three GIF files showing you the product as well as some cool buttons.

Client: This is great. I'm going to fill in all the blanks with my name, address, phone, and email and click on the submit button. (psst, when you get all my information please run the script named order.php.

Server: Okee Dokee. Let's see (rummaging through a shoebox filled with CGI scripts and PHP pages.) Ah, here it is, order.php. I'll just feed all your information into the script and see what happens.....

Ok, first the script saved all your order information in a database.
Then it built a new web page for you.
It looks like a verification page of your order.
Here you go. No graphics with this web page. Maybe next time.

Client: Oh good, just what I wanted. I'm going to print this out so I have a hard copy for my files. I'm off to look somewhere else on the web.

(Notice the client doesn't bother to respond to the server. The server is busy handling other requests and doesn't require a response from the client. It considered its job done when it responded with the verification.)

Note: The graphic shows CGI scripts. CGI stands for Common Gateway Interface. A CGI script is the interface between the client and the server or gateway. These are commonly written using Perl. Other options on the server include web pages with PHP code embedded in with the XHTML elements.

Server-Side Scripting

Server-side scripting allows the programmer to handle information from the client and run programs that will create new web pages on the fly. The benefit of this is that the programmer maintains control and the code is more secure.

Client-side languages, such as JavaScript respond differently on different browsers (you never know what your user will be viewing the web pages with) and the source-code is very visible to anyone that can see the web page.

Some of the more common server-side scripting languages include:

  • PHP - A very popular as a scripting language.
  • ASP.NET - Active Server Pages running on Microsoft servers.
  • JSP - Java Server Pages utilize the Java Language (the big brother to JavaScript)
  • Perl - one of the original languages used to create CGI scripts..
  • ColdFusion by Adobe is similar to PHP and ASP.NET and requires a special ColdFusion server.

All of these languages run on the web server and handle the form data as it is submitted by the user.

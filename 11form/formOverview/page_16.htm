Typing in Test Data

When testing programs you will spend a lot of time testing something over and over.

It is very common for a developer to test a page 20 - 30 times getting everything just right. If that developer takes 60 seconds to input the test data each time, they will end up wasting over 20 - 30 minutes just typing in data!

When you are working on a tight deadline or trying to finish up a project in the small hours of the morning, 20 minutes is a lot of time!

Here's some tips that will save time and make your debugging easier:

  1. Set up default values in the form when testing out the web page.
  2. Keep things short. Instead of typing in a full name "John Jacob Jingle Himmer Schmidt", just type in the letter "a" or "first".

    For multiple lines use "programming code" by hitting keys at random. Make sure to include lots of spaces so the line will word wrap: j klfjdj alksjfdj dfl jadflja dfk j ljdfja fjfljsalfjsfdj jf jadfl jlsdjf lsjfl jf fjlfdjlajfl lfdj ljj'jf lkj flj f.
  3. Use smart data. Input data that matches the field.
    A zip field might be "1111"
    A state could be "MN".
    A first name field might be "first".
    Phone numbers can be (123) 456-7890.

    Long numbers like serial numbers can be handled the same way: 1234567890123.
  4. Keep track of which test is which. If you have to test something 10-20 times (not uncommon) it will be easier to track the results if you add it to your data. First name could be "name1", "name2", and "name3" for each consecutive trial.
  5. Identify test data. Sometimes test data gets mixed up with live data. Always include phrases like "testing" or "testTextAreaj lkjd fljfljdj ldj fjdfl jkljflj etc." or "testStreet", "testCity", "testZip". If the data does get mixed in with real data it is easy to search and destroy.
  6. Be consistent with your test data. Sometimes it is the data that breaks a program. Always use the same test data over and over so you know you are testing the form or program and not the data.
  7. If things are not changing on the web page make sure you are viewing the correct file. Often you may end up editing one file and the browser is viewing another file. (I won't tell you how often this has happened to me, and I'm a paid professional.) In an very obvious spot, like at the end of an <h1> text add something weird like "ABC" and then view the page. If your new text does not display than check to make sure you are viewing the same file that you are editing.

The faster and more consistent you can input test data, the less time it will take to debug your program or form.

Web Forms - An Overview

Client/Server Architecture

Before you learn the elements that make up forms and their objects let's go over the client/server model.

Remember that web pages "live" on a web server. A request is made from a client machine requesting a certain page from the web server. When the server receives the request it sends the necessary files back to the client and the web page displays in the browser window.

There are all types of files on the web server. Not only the HTML files (plain text) but image files as well. There are also small programs on the server called server-side scripts.

Just as the web page knows which image files should be displayed, the <form> element knows the name of the server-side script that should run when the form is submitted by the user.

When the user clicks on the "submit" button, the information from the form is sent to the server along with the name of the program, or script, that will handle the data.

A server-side script can do all types of things. A few examples are:

  • store the data in a database,
  • send an email to a specific address,
  • pull new data from a database (based on the form data criteria)
  • create a new web page

In this course you will only be learning how to create a form. You won't be able to manipulate data on the server because you won't have the ability to create the custom server-side scripts. Those techniques will be covered in courses pertaining to JavaScript, PHP and other scripting languages or server-side programming.

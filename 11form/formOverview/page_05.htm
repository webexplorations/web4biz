Web Forms - An Overview

Objects to Put in the Form

There are 9 different controls that can be used inside the <form> element.

The Button, Reset, and Submit all make buttons and allow the web page to capture clicks from the user. The Button is a general purpose control or object while the Reset button clear's the values from all of the form objects. The Submit button's only purpose is to send the data in the form to the server.

Checkboxes are used to find user preferences when there can be more than one choice.

Radio buttons are used when there can only be one choice. They are named for the old fashioned car radios that allowed a person to select preset stations by pushing a row of buttons.

List boxes or drop-down boxes allow the user to select a choice (or several choices from a list). An example would be a list of state abbreviations.

Text, Password, and TextArea allow the user to type in text information. Text box is usually for short answers such as name, address, and email. Password text boxes display the input as a series of asterisks (*) to help keep passwords secure from on-lookers. Textarea allows the user to type in multiple lines of information. A comment box would be a good use for a textarea object.

The Hidden object is used to send information to the server that the end-user will not see. For example, sending the name of a web site that the server can use when generating an email. The web site is included in the subject line so the administrator or client can see where the email came from.

Each of these has a prefix that makes it easier to identify what it is, especially when you are writing code that runs out on the server. It makes a lot more sense knowing that you are working with a txtName text box than "name" which could be a text area, a radio button, or a drop down list.

Use the prefixes consistently with all your forms and your programming efforts will be much simpler in the future.

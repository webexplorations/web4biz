Web Forms - An Overview

Form action="mailto: "

The submit button now brings us back full circle: to the <form> tag. When the submit button is clicked, it uses the action=" " attribute and method=" " attribute in the <form> tag to determine how the form values should be sent to the server and what file should handle the form information out on the server.

, and how the data from the form should be transmitted to the server.

action=" " The action attribute is the name of the file on the web server which will read and handle the form values.

action=mailto:yourClientEmail@gmail.com You can also use the mailto: protocol to simulate using a script. When a browser sees mailto: it automatically opens up the email client program that is installed on the user's computer. (This only works when an email client is installed and will not work on computers, such as the school labs, that do not have a mail program installed.)

method=" " The method attribute tells the browser how to send the form values to the server. The values for this attribute can be either "post" or "get".

method="post" Post has the browser pack up the form values into a bundle, much like packing a suitcase full of data and shipping it over the Internet to the server. This "suitcase" of information can be encrypted if necessary.

method="get" Get takes the form values and adds them to the end of the URL request that is sent to the server. Using the get method is not secure and any information from the form can be easily seen by anyone that cares to capture the information. The is also a character limit to the get method.

So, let's do something with the form values by adding the necessary attributes to the <form> tag. Use your own email address to test everything out.

When you click the submit button your email program will run.

And, because we use the method get you can see the values on the URL being sent to the server (in this case the mailto: email program)

enctype="text/plain" The enctype=" " attribute ensures that the data will be in a readable (line by line) format in the email that is sent.

If the enctype=" " is not used the data will be sent in the email as a clump of text. All the spaces are replaced with plus signs "+" and each variable is separated from the others with an ampersand "&".

If you use the attribute enctype="text/plain" each form value pair will be on a separate, easy-to-read line.

Writing To Standards

The world changed in 1913 when Henry Ford fine-tuned the assembly line process (which was originally devised by Eli Whitney in 1799).

[D]Instead of lots of customized pieces being slowly and painstakingly pieced together, parts were standardized and assembled as the autos moved along a conveyor belt. Ford was able to make twice as many cars as all his competitors combined and his cost savings allowed him to cut the price in half and then in half again.

Think where we would be if each railroad company in the United States had used a different size railroad track! Or, think of the problems that would result if every neighbourhood had different types of connections for their fire hydrants. What if each grocery store used their own set of weights and measures. And can you imagine how troublesome it was to travel in Europe when each country had its own currency, before the Euro was adopted.

Now, imagine if every browser used a different set of HTML codes to display the pages. This really happened during the 1990's when Netscape (which later became Mozilla) and Microsoft's Internet Explorer (IE) were competing for market share. Programmers had to write one set of code for people using the Netscape browser and another set of code for people viewing the page using IE.

Tim Berners-Lee saved the day by forming the W3C (World Wide Web Consortium). The W3C is a group of companies (including Netscape and Microsoft) that sit down and agree on a standard way of doing things.

There are several HTML standards as well as the newer HTML standard. You can find them out on the W3C site. Here is the HTML 1.1 standard (http://www.w3.org/TR/HTML-basic/)

What happens if you don't follow these standards?

Someone often says, "I write HTML code anyway I like and it shows up fine in my browser."

That may be true, but what if your customer is using a different browser? Often, a web page that works in Internet Explorer will show up as a white screen of nothing in Netscape or Opera. Once the code is rewritten to the standard, everything displays fine. So, which way do you want it? Your way, or so everyone can use the pages you create? (The answer sounds something like this: ka ching. Your client could lose customers, i.e. money, if they can't view the web pages.)

Also, by following standards you know your web pages will be viewable on devices and programs that haven't even been invented yet. This is especially true if you adapt the newer HTML standards.

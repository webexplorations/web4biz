Web Development - Tables

The Basic Table Elements

There are four basic table elements.

<table> is the main table element, holding all the others.

<tr> sets up a row inside the table. Each <tr> element will contain a series of column or header tags.

<th> sets up a series of header columns. The text in these columns is automatically made bold and is centered by the browser. You can also change the appearance of these columns using CSS.

<td> sets up a series of columns to hold text and/or data.

IMPORTANT! Don't forget the stop tags. It is recommended that each time you type in a new tag, immediately add in its matching stop tag. Missing stop tags are the most common problem for people adding tables to a web page.

Open up a basic HTML file and let's create a table.

(1) First add the <table> start and stop tags. By default browsers display tables without borders so use the border=" " attribute to add a 1 pixel border.
<table border="1">

(2) Add a row element using the <tr> element. Type in the </tr> right away. You may also want to add a comment on the line above the start tag designating what the row does. Tables have a lot of tags and consistent commenting, especially separating the rows, will greatly help you maintain the page in the future.

(3) Add two or three column headings using the <th> element inside the <tr> tags. Use smart data designating each column so they are easy to identify.

(4) Create another row and fill it with column data using the <td> element. Here's a short cut: Highlight the first row and copy and paste it. Then all you have to do is change the th to td and modify the contents appropriately. Once you have one row built, filled with <td> columns it's a simple matter to copy it and paste it below, simply changing the data.

Use smart data. In this example you can quickly see the location of each column and row by the information contained in each cell. Later on, when you have everything working correctly you can easily replace this smart data with the actual client data.

(Remember you are building a basic table template file that will be edited to create other web pages. The smart data will be acting as a place holder for whatever data is put in each table.)

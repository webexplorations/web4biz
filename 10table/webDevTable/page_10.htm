If Cell Width Doesn't Change

Often a table column won't "collapse" or go smaller even though the browser window size is reduced. Here's some trouble-shooting tips to help you fix this problem.

(1) Keep in mind that the widest cell determines how big all the cells are in that column. For example, you may have a large image hidden "below the fold" that you don't see. It will determine how wide all the cells are in that column.

(2) Extra lines <br /> are often introduced when you copy text into a table cell. Position the cursor just to the right of the last character is a cell and use the delete key to remove all the extra spaces or XHTML characters.

(3) Paragraphs can add a lot of depth to text. Go in and remove all the <p> elements from inside your cells to help them collapse.

(4) If you are using DreamWeaver these changes will not go into effect immediately when in design mode. Often you have to resize the view window in order to get a table to collapse to its true size. When in doubt, preview in a browser. Don't trust DreamWeaver's rending of a page.

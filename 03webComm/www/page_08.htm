DNS - Domain Name Server

[D]

Let's request a web page. Let's request a search from Google.com.

Every web site has its own unique address. Its IP address, a 12-digit number. So, just type in Google's IP address in the "address" or URL field of your browser.

What? You don't know the IP address?

Ok then, let's put the DNS (Domain Name Server) to work.

Type in the words "google.com" in the address (URL) field of your browser.

This will send a request to your ISP which immediately notices that you are sending string characters not numbers.

Your ISP uses its DNS - Domain Name Server to lookup the name you typed in and get the IP address. It's almost like a phone book. You look up a name and come up with the unique phone number for that person.

Your ISP sends this IP address back to your browser which then sends out another request asking for 216.239.33.100, which used to be the IP address for the domain named Google.com.

Without the DNS we would be stuck having to type in the IP address for every web site we wanted to see. Not a fun idea for people that prefer using words over numbers.

The DNS isn't such a big deal until you suddenly remember that there are billions of servers that make up the Web. That's a lot of IP addresses and domain names to keep track of!

The DNS system works in zones.

The first DNS zone is on your computer. It consists of a file named hosts. In Windows XP this file is located in: C:\WINDOWS\system32\drivers\etc\hosts.

Looking at this file you will see that it is made up of three columns. The first shows the IP addresses, a second column contains the domain names, and an optional third column for comments:

127.0.0.1 localhost # this is the IP address for your computer

If the domain you typed in is not listed here, your browser goes out to your ISP and uses their DNS list. If your ISP can't find a match they go out on the Domain Name System to a higher zone. Your ISP and other zones cache, or hold, new names that are discovered, keeping the system fairly dynamic.

Once a match is found the resulting IP address is sent back to your browser. If a match can't be found you receive the infamous "Server not found" error.
